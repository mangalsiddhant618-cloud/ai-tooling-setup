# What good prompt tracking looks like in practice (B2B SaaS, CRM category)
**Kevin Indig** · [View post](https://www.linkedin.com/posts/kevinindig_good-prompt-tracking-starts-with-sample-design-activity-7470197035565002752-T8ow)

## Post text
Good prompt tracking starts with sample design.

For your average B2B SaaS CRM category, I’d start with ~40 seed prompts: 12 brand, 12 category, and 16 problem prompts. The problem prompts matter because that is where purchase intent usually lives.

Then I’d run each prompt 5x per platform every week. ChatGPT, Perplexity, Gemini, and Google AI Overviews should each get their own score because aggregation hides how differently the engines behave.

The panel also needs personas. CFO, IT, and marketing buyers evaluate the same CRM category through different criteria, so averaging them together hides where a brand actually wins or loses.

The metric set should include mention rate, citation rate, average position, sentiment, and the attributes attached to each mention, all with confidence intervals.

That is how prompt tracking becomes more accurate: each answer becomes one sample in a designed panel, not one truth in a dashboard.

## Infographic (transcribed) — "What good prompt tracking looks like in practice"
*Worked example: B2B SaaS, CRM category*
1. **Prompt set — 40 seed prompts:** 12 brand, 12 category, 16 problem prompts, weighted toward where purchase intent lives.
2. **Platforms — track each engine separately:** ChatGPT, Perplexity, Gemini, and Google AI Overviews each get their own score.
3. **Run config — 5 reps per prompt, every week:** repeated runs estimate the confidence band instead of reporting one answer as truth.
4. **Personas — 3 buyer panels:** customize category and problem prompts for CFO, IT, and marketing buyers.
5. **Metrics — report accuracy, not raw rank:** mention rate, citation rate, average position, sentiment, and brand attributes, with confidence intervals.

## Why this matters
A rigorous, copyable measurement design for AI-search visibility — the missing
"how do we actually track this?" layer most AEO advice skips. Key principles for a
production playbook: (1) weight prompts toward problem/purchase-intent queries;
(2) score each engine separately (they behave differently); (3) run repeated samples
to get confidence bands, not single-shot "truth"; (4) segment by buyer persona; and
(5) report mention/citation/sentiment with confidence intervals rather than a vanity
rank. Treats LLM answers as samples in a designed panel — proper experiment design,
not dashboard theater.
